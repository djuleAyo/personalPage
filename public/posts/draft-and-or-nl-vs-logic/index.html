So if I say

I assume both are true in some kind of way. Here we can deduce we are referring an image of a beach. Image is not a beach but is associated with it since it is a picture of it. Some significant logic jump is made here, but non the less initial assumption is that both are true is met in a bit broader sense.

Yet if I ask who will sit at that table? And you say:

Marco Luca Natasha

This is set enumeration, not conjuction. Set enumeration is in fact disjunction. As it is well known in mathematics, without any dilemma, as set theory is the basis of it.

x∈{a,b,c}≡(x=a)∨(x=b)∨(x=c)

While anyone who studies logic for just a bit knows those are almost like opposites. Disjunction acts as union and conjuction as cutting. So how come we use same syntax for both? Well see this implication

A & B => A | B

It is valid and very much the basis of said overloading. So when I say

Marco Luca Natasha

I mean set enumeraiton, I mean any of them can sit at the table. More then that. All of them can sit at the table. And if all do, well, some of them will sit there. And that is justification for the overloading.

Now NLP, AI and those are hot topics. They are frontend of user centric systems. And the fact UX is becoming more user centric is good. So any humane system should. And there fore analyzing the limits of minimal syntax in case of simle thing like conjuction and disjunction is important.
